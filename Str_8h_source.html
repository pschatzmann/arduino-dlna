10/*
11 * @brief Heap-backed string utility used throughout tiny_dlna when
12 * DLNA_USE_STD is enabled.
13 *
14 * This class intentionally uses composition around `std::string` and
15 * forwards a small, controlled subset of operations used by the codebase.
16 * It attempts to preserve the historical `Str` API (length as `int`,
17 * `add`/`set` helpers, substring helpers, simple trimming and search
18 * utilities) while keeping ownership explicit and avoiding exposure of
19 * the entire `std::string` interface.
20 *
21 * Usage notes:
22 * - Most callers treat `Str` similarly to the original project string
23 * type; it provides an implicit conversion to `const char*` for
24 * convenience when interoperating with legacy `StrView` APIs.
25 * - The class is intended for embedded-friendly usage where a heap
26 * buffer is acceptable; `isOnHeap()` returns true for the callers that
27 * check allocation semantics.
