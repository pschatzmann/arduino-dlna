9/*
10 * @brief Heap-backed string utility used throughout tiny_dlna
11 *
12 * This class intentionally uses composition around `std::string` and
13 * forwards a small, controlled subset of operations used by the codebase.
14 * It attempts to preserve the historical `Str` API (length as `int`,
15 * `add`/`set` helpers, substring helpers, simple trimming and search
16 * utilities) while keeping ownership explicit and avoiding exposure of
17 * the entire `std::string` interface.
18 *
19 * Usage notes:
20 * - Most callers treat `Str` similarly to the original project string
21 * type; it provides an implicit conversion to `const char*` for
22 * convenience when interoperating with legacy `StrView` APIs.
23 * - The class is intended for embedded-friendly usage where a heap
24 * buffer is acceptable; `isOnHeap()` returns true for the callers that
25 * check allocation semantics.
26 */
