11/*
12 * @class Vector
13 * @brief Lightweight wrapper around std::vector with Arduino-friendly helpers and a pluggable allocator.
14 *
15 * This container inherits from std::vector to preserve its API and complexity guarantees while
16 * adding a few conveniences tailored for embedded/Arduino targets:
17 *
18 * - Uses a configurable allocator (default: ALLOCATOR<T>) defined by the library to enable
19 * custom allocation strategies or tracking.
20 * - Keeps all standard std::vector constructors, types, and methods available.
21 * - Exposes std::vector erase overloads and provides index-based helpers.
22 * - Adds reset() which performs clear() followed by shrink_to_fit() to aggressively release memory.
23 *
24 * Template parameters:
25 * - @tparam T Element/value type stored in the vector.
26 * - @tparam Alloc Allocator type (defaults to ALLOCATOR<T> as configured in dlna_config.h).
27 *
28 * Differences vs std::vector:
29 * - erase(size_t index): convenience overload that erases an element by index and returns the
30 * iterator following the erased element (or end() if the last element was erased). If the index
31 * is out of range, end() is returned and no action is taken.
32 * - eraseIndex(int index): bounds-checked helper; does nothing on out-of-range indices.
33 *
34 * Complexity: Matches std::vector for the corresponding operations (e.g. erase is linear in the
35 * number of moved elements). Reserve/capacity/iterator invalidation rules are also identical to
36 * std::vector.
37 */
