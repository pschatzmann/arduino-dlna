◆ MediaEvent

enum class tiny_dlna::MediaEvent
strong

Events emitted by the MediaRenderer to notify the application about playback and rendering control changes.

The application registers a callback using setMediaEventHandler() and will receive the event along with a reference to the emitting MediaRenderer instance. The handler should query the renderer (for example getCurrentUri(), getVolume(), getMime()) to determine the appropriate action for each event.

Events:

  • SET_URI: The renderer received a new URI (use getCurrentUri()).
  • PLAY: Start or resume playback of the current URI.
  • PAUSE: Pause playback.
  • STOP: Stop playback and reset playback state.
  • SET_VOLUME: The volume was changed (use getVolume()).
  • SET_MUTE: The mute state changed (use isMuted()).
